GUIA DE REFERÊNCIA RÁPIDA

GIT

Comandos Essenciais para Controle de Versão

CONFIGURAÇÃO INICIAL

Comando Descrição Exemplo
git config --global user.name Define seu nome de usuário git config --global user.name "Seu Nome"
git config --global user.email Define seu email git config --global user.email "seu@email.com"
git config --list Lista todas as configurações git config --list

INICIAR REPOSITÓRIO

Comando Descrição Exemplo
git init Inicializa um repositório Git local git init
git clone Clona um repositório remoto git clone https://github.com/usuario/repo.git

TRABALHANDO COM ALTERAÇÕES

Comando Descrição Exemplo
git status Mostra o estado dos arquivos git status
git add Adiciona arquivos à staging area git add .
git commit Registra as alterações git commit -m "Mensagem"
git commit -am Adiciona e commita arquivos rastreados git commit -am "Mensagem"

VER HISTÓRICO E DIFERENÇAS

Comando Descrição Exemplo
git log Exibe o histórico de commits git log
git log --oneline Histórico compacto git log --oneline
git diff Mostra diferenças entre arquivos git diff
git diff --staged Mostra diferenças na staging area git diff --staged

BRANCHES (RAMIFICAÇÕES)

Comando Descrição Exemplo
git branch Lista todas as branches locales git branch
git branch [nome] Cria uma nova branch git branch nova-feature
git checkout [nome] Muda para uma branch existente git checkout main
git checkout -b [nome] Cria e muda para uma nova branch git checkout -b nova-feature
git merge [nome] Mescla uma branch à branch atual git merge nova-feature
git branch -d [nome] Deleta uma branch local git branch -d branch-antiga

REPOSITÓRIO REMOTO (GITHUB)

Comando Descrição Exemplo
git remote add origin Adiciona um repositório remoto git remote add origin [url]
git push -u origin Envia commits para o remoto git push -u origin main
git push Envia commits (após primeiro push) git push
git pull Puxa alterações do remoto git pull
git fetch Busca alterações sem merge git fetch

DESFAZENDO COISAS

Comando Descrição Exemplo
git restore [arquivo] Descarta alterações não commitadas git restore arquivo.txt
git reset --hard HEAD Remove todas as alterações não commitadas git reset --hard HEAD
git reset --soft HEAD^ Desfaz o último commit, mantendo alterações git reset --soft HEAD^
git revert [hash] Cria novo commit desfazendo alterações git revert a1b2c3d

OUTROS COMANDOS ÚTEIS

Comando Descrição Exemplo
git stash Armazena temporariamente alterações git stash
git stash pop Recupera alterações armazenadas git stash pop
git rm [arquivo] Remove arquivo do repositório git rm arquivo.txt
git mv [antigo] [novo] Renomeia ou move arquivo git mv antigo.txt novo.txt
git tag Gerencia tags para marcação de versões git tag -a v1.0 -m "Versão 1.0"
git rebase Reaplica commits em outra base (avançado) git rebase main

FLAGS MAIS USADAS

Flag Descrição Uso
-m Adiciona mensagem ao commit git commit -m "Mensagem"
-a Adiciona automaticamente arquivos rastreados git commit -am "Mensagem"
-u Define upstream (conexão com remoto) git push -u origin main
-b Cria e muda para uma nova branch git checkout -b nova-branch
-d Deleta branch com segurança git branch -d branch-antiga
-D Força exclusão de branch git branch -D branch-perigosa

FLUXOS DE TRABALHOS TÍPICOS

Configuração Global do Git: Define sua identidade para todos os repositórios
  1. git config --global user.name "Seu Nome" - Define seu nome para todos os commits
    Este nome aparecerá em todos os seus commits. Use seu nome real ou nickname profissional.
  2. git config --global user.email "seu@email.com" - Define seu email para todos os commits
    Use o mesmo email associado à sua conta do GitHub/GitLab. Isso conecta seus commits à sua conta.
  3. git config --global --list - Verifica todas as configurações globais
    Comando útil para confirmar que suas configurações foram salvas corretamente.
git Clone: Copia um repositório remoto para sua máquina local
  1. git clone [url] - Clona um repositório existente
  2. cd [nome-do-repositorio] - Entra na pasta do repositório
  3. Configure usuário e email (apenas se não tiver configurado globalmente ou se preferir configurar de forma diferente):
    • git config user.name "Seu Nome"
    • git config user.email "seu@email.com"
git Init: Inicializa um novo repositório Git local
  1. mkdir [nome-do-projeto] - Cria uma pasta para o projeto
  2. cd [nome-do-projeto] - Entra na pasta do projeto
  3. git init - Inicializa o repositório Git
  4. Configure usuário e email (apenas se não tiver configurado globalmente ou se preferir configurar de forma diferente):
    • git config user.name "Seu Nome"
    • git config user.email "seu@email.com"
  5. git remote add origin [url] - Conecta ao repositório remoto (opcional)
Trabalhando na Branch Main: Desenvolvimento direto na branch principal
  1. git status - Verifica o estado do repositório
  2. git add . - Adiciona todas as alterações
  3. git status - Confirma as alterações adicionadas
  4. git commit -m "Mensagem descritiva" - Commita as alterações
  5. git push -u origin main - Envia para o repositório remoto
Trabalhando em uma Branch: Desenvolvimento isolado em uma branch separada
  1. git checkout -b [minha-branch] - Cria e muda para uma nova branch
  2. git status - Verifica o estado do repositório
  3. git add . - Adiciona todas as alterações
  4. git status - Confirma as alterações adicionadas
  5. git commit -m "Mensagem descritiva" - Commita as alterações
  6. git push -u origin [minha-branch] - Envia para o repositório remoto
Mesclando Branches: Integrando changes de uma branch para outra
  1. git checkout main - Muda para a branch main
  2. git pull origin main - Atualiza a branch main local
  3. git merge [minha-branch] - Mescla a branch na main
    • Se houver conflitos, resolva-os e depois faça git add . e git commit
  4. git push origin main - Envia as changes mescladas para o remoto
  5. git branch -d [minha-branch] - Deleta a branch local (opcional)
  6. git push origin --delete [minha-branch] - Deleta a branch remota (opcional)
Atualizando uma Branch com Changes da Main: Mantendo sua branch atualizada
  1. git checkout [minha-branch] - Muda para sua branch
  2. git fetch origin - Busca as changes do repositório remoto
  3. git merge origin/main - Mescla as changes da main na sua branch
    • Alternativa: git rebase origin/main para reaplicar suas changes sobre a main atualizada
  4. Resolva quaisquer conflitos se necessário
  5. git push origin [minha-branch] - Atualiza a branch remota
Verificação de Configurações do Git: Comandos para verificar suas configurações atuais
  1. git config user.name - Mostra o nome de usuário configurado para este repositório
    Se retornar vazio, significa que não há configuração específica para este repositório e será usada a configuração global.
  2. git config user.email - Mostra o email configurado para este repositório
    Se retornar vazio, significa que não há configuração específica para este repositório e será usada a configuração global.
  3. git config --list - Mostra TODAS as configurações do Git
    Exibe todas as configurações, incluindo globais, do sistema e específicas do repositório atual.
  4. git config --global --list - Mostra apenas as configurações globais
    Útil para verificar quais configurações estão definidas globalmente em sua máquina.
  5. git config --local --list - Mostra apenas as configurações do repositório atual
    Exibe apenas as configurações específicas do repositório em que você está trabalhando.
Hierarquia de Configuração: Entenda a ordem de prioridade das configurações
  1. Local (por repositório) - Maior prioridade: .git/config
  2. Global (por usuário) - Segunda prioridade: ~/.gitconfig
  3. System (para todo o sistema) - Menor prioridade: /etc/gitconfig
Fluxo de Desfazer Alterações: Como reverter mudanças indesejadas
  1. git checkout -- [arquivo] - Descarta alterações não commitadas em um arquivo
  2. git reset --hard HEAD - Descarta TODAS as alterações não commitadas
  3. git revert [hash-do-commit] - Cria um novo commit que desfaz um commit anterior
  4. git reset --soft HEAD~1 - Desfaz o último commit, mantendo as alterações
Fluxo de Trabalho com Stash: Como guardar alterações temporariamente
  1. git stash - Guarda as alterações atuais em uma área temporária
  2. git stash list - Lista todos os stashes guardados
  3. git stash apply - Recupera as alterações do stash mais recente
  4. git stash drop - Remove o stash mais recente
  5. git stash pop - Aplica o stash mais recente e o remove da lista
Fluxo de Visualização de Histórico: Como explorar o histórico do projeto
  1. git log - Mostra o histórico de commits
  2. git log --oneline - Mostra o histórico resumido em uma linha por commit
  3. git log --graph - Mostra o histórico com um gráfico de branches
  4. git blame [arquivo] - Mostra quem modificou cada linha de um arquivo
  5. git diff - Mostra as diferenças entre arquivos modificados e sua versão commitada
Fluxo de Synchronização com Repositório Remoto: Como manter seu repositório atualizado
  1. git fetch origin - Baixa as alterações do remoto, mas não aplica no seu branch
  2. git pull origin [branch] - Baixa e mescla as alterações do branch remoto
  3. git push origin [branch] - Envia seus commits para o branch remoto
  4. git remote -v - Lista os repositórios remotos configurados
  5. git remote show origin - Mostra informações detalhadas sobre o remoto 'origin'
Fluxo de Resolução de Conflitos: Como resolver conflitos de merge
  1. git status - Identifica arquivos com conflitos
  2. Abra os arquivos com conflitos e resolva manualmente (procure por marcadores como <<<<<<<)
  3. git add [arquivo] - Marca os arquivos como resolvidos
  4. git commit - Finaliza o merge após resolver todos os conflitos
Fluxo de Trabalho com .gitignore: Como ignorar arquivos e pastas desnecessários
  1. Crie um arquivo chamado .gitignore na raiz do seu repositório
  2. Adicione os padrões de arquivos/pastas que deseja ignorar:
    • # Ignorar arquivos de log
    • *.log
    • # Ignorar pasta node_modules
    • node_modules/
    • # Ignorar arquivos de ambiente mas manter exemplo
    • .env
    • !.env.example
  3. Salve o arquivo .gitignore
  4. git add .gitignore - Adiciona o arquivo de ignore ao repositório
  5. git commit -m "Adiciona arquivo .gitignore" - Commita as alterações
  6. Os arquivos/pastas correspondentes aos padrões serão ignorados pelo Git
Fluxo de Versionamento com Tags: Como marcar versões do seu projeto
  1. git tag - Lista todas as tags existentes
  2. git tag -a v1.0.0 -m "Versão 1.0.0" - Cria uma tag anotada para a versão
    Use versionamento semântico (MAJOR.MINOR.PATCH) para nomear suas tags.
  3. git push origin v1.0.0 - Envia a tag para o repositório remoto
    Por padrão, o git push não envia tags. É necessário enviá-las explicitamente.
  4. git tag -d v1.0.0 - Deleta uma tag local (se necessário)
  5. git push --delete origin v1.0.0 - Deleta uma tag remota (se necessário)
  6. git checkout v1.0.0 - Para visualizar o código em uma versão específica
Fluxo de Rebase (Avançado): Como reorganizar o histórico de commits
Atenção: O rebase altera o histórico. Não use em branches compartilhadas onde outras pessoas estão trabalhando!
  1. git checkout minha-branch - Mude para a branch que deseja reorganizar
  2. git rebase main - Reaplica os commits da sua branch sobre a main atualizada
    Isso coloca seus commits no topo da branch main, criando um histórico linear.
  3. Se houver conflitos durante o rebase:
    • Resolva os conflitos nos arquivos
    • git add . - Marca os conflitos como resolvidos
    • git rebase --continue - Continua o processo de rebase
  4. git push --force-with-lease origin minha-branch - Atualiza a branch remota
    O --force-with-lease é mais seguro que --force, pois verifica se há alterações remotas que você não possui localmente.
  5. Se quiser cancelar um rebase em andamento: git rebase --abort
Boas Práticas com Git: Diretrizes para um uso eficiente e seguro
  1. Commits atômicos: Cada commit deve representar uma única alteração lógica
  2. Mensagens claras: Escreva mensagens de commit descritivas no imperativo
    • Ruim: "mudanças" ou "fix"
    • Bom: "Adiciona funcionalidade de login" ou "Corrige cálculo de impostos"
  3. Commit cedo e frequentemente: Faça commits pequenos e regulares
  4. Use branches para features: Isole novas funcionalidades em branches separadas
  5. Antes de fazer push: Sempre execute testes e verifique se tudo funciona
  6. Mantenha seu repositório atualizado: Use git fetch regularmente para acompanhar alterações remotas
  7. Use .gitignore: Ignore arquivos desnecessários (logs, dependências, arquivos de ambiente)
  8. Documente: Mantenha um README atualizado com informações do projeto

CONCEITOS IMPORTANTES

Repositório

Local onde o Git armazena todos os arquivos e histórico do projeto

Commit

Snapshot (foto) do seu projeto em um determinado momento

Branch

Ramificação independente do projeto para desenvolver features

Merge

Ação de unir o conteúdo de duas branches

Pull Request

Solicitação para incorporar alterações de uma branch em outra (GitHub)

Staging Area

Área intermediária onde preparamos arquivos para commit

Working Directory

Área de trabalho onde você modifica os arquivos do projeto

HEAD

Ponteiro que referencia o commit atual em que você está trabalhando

Remote

Repositório remoto (como GitHub/GitLab) onde o código é hospedado

Clone

Cópia local de um repositório remoto com todo seu histórico

Fork

Cópia pessoal de um repositório de outro usuário no GitHub

Conflict

Conflito que ocorre quando o Git não pode mesclar alterações automaticamente

Stash

Local temporário para armazenar alterações não commitadas

Tag

Marcador estático que aponta para um commit específico (usado para versões)

Rebase

Alternativa ao merge para integrar changes de uma branch em outra

Origin

Nome padrão dado ao repositório remoto principal quando você clona

Upstream

Referência ao repositório original de onde seu fork foi criado

Hook

Scripts personalizados que são executados em eventos específicos do Git

.gitignore

Arquivo que especifica quais arquivos/diretórios o Git deve ignorar

Dica: Sempre use git status para verificar o estado do seu repositório antes e depois de executar comandos.
Aviso: Comandos com --hard, -D ou -f podem causar perda de dados. Use com cuidado!
Perigo: git rebase e git push --force alteram o histórico do Git. Nunca use em branches compartilhadas com outras pessoas!
